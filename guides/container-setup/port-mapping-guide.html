Overview

The container exposes common development ports, allowing you to run web applications inside the container and access them from your host browser.

Exposed Ports

Frontend Development

Port Framework/Tool Usage
3000React, Next.jsDefault dev server
3001AlternativeSecondary frontend app
5173ViteDefault Vite dev server
8080Vue.jsDefault Vue dev server
4200AngularDefault Angular dev server

Backend Development

Port Framework Usage
5000FlaskDefault Flask dev server
8000Django, FastAPIDefault server
8001AlternativeSecondary backend
9000AlternativeAdditional backend

Important: Binding to 0.0.0.0

Correct:

app.run(host='0.0.0.0', port=5000)

Wrong:

app.run(host='localhost', port=5000)  # Won't be accessible from host!

Quick Examples

React Application

# Inside container
cd workspace/my-react-app
npm create vite@latest . -- --template react
npm install
npm run dev

# Access from host: http://localhost:5173

Flask API

# Inside container
cd workspace/my-api
python app.py  # Must bind to 0.0.0.0

# Access from host: http://localhost:5000

Next.js Application

# Inside container
cd workspace/my-nextjs-app
npm run dev

# Access from host: http://localhost:3000

Best Practices

  • Always bind to 0.0.0.0 (not localhost)
  • Access from host via http://localhost:PORT
  • Multiple apps can run on different ports
  • Check port conflicts with netstat -tlnp
  • Add custom ports in docker-compose.yml if needed
